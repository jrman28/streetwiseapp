# Neighborhood One-Pager — Replit Build Plan

## BUX vs. The Lighter Stack

You're right. BUX is a 24/7 VPS with a full Claude Code agent + persistent browser. That's a whole server for what is essentially: "scrape 5 websites, format a PDF, email it."

Here's the swap:

| BUX Does | Replacement | Why it's better for this |
|---|---|---|
| Persistent browser | **Firecrawl** (scrape + interact) | API call, no infra to manage. Handles anti-bot, JS rendering, proxies. |
| 24/7 VPS running tasks | **Trigger.dev** (scheduled jobs) | Serverless cron. Runs your pipeline on a schedule. Free tier = 10K runs/mo. |
| Telegram delivery | **Resend** (email) | Agents want email, not Telegram. $0 for first 3K emails/mo. |
| CAPTCHA solving | **Firecrawl** or **Browser Use Cloud** | Both handle CAPTCHAs natively in their cloud infra. |

### Firecrawl vs. Browser Use Cloud — When to Use Which

| Task | Use Firecrawl | Use Browser Use Cloud |
|---|---|---|
| Scrape a page and get clean markdown | ✅ `firecrawl.scrape(url)` | Overkill |
| Search the web for a query | ✅ `firecrawl.search(query)` | Overkill |
| Click through a multi-step flow | ❌ | ✅ `client.run("Go to X, click Y, extract Z")` |
| Extract structured data from a known page | ✅ with `extract` or `interact` | Either works |
| Handle a login wall | ❌ | ✅ (persistent sessions, CAPTCHA solving) |

**For the Neighborhood One-Pager, Firecrawl does 90% of the work.** You only need Browser Use Cloud if a data source requires multi-step interaction (clicking through filters, etc.).

## Core Code Sketch (TypeScript)

```typescript
// scrapers/schools.ts
import Firecrawl from '@mendable/firecrawl-js';

const firecrawl = new Firecrawl({ apiKey: process.env.FIRECRAWL_API_KEY });

export async function getSchools(zip: string) {
  const result = await firecrawl.scrape(
    `https://www.greatschools.org/search/search.page?zip=${zip}`,
    { formats: ['markdown'] }
  );
  // Parse the markdown for school names + ratings
  return parseSchools(result.data.markdown);
}
```

```typescript
// scrapers/sentiment.ts  
export async function getNeighborhoodSentiment(neighborhood: string, city: string) {
  // Option A: Firecrawl search (simpler, no API keys beyond Firecrawl)
  const results = await firecrawl.search(
    `${neighborhood} ${city} neighborhood living reddit`,
    { limit: 10 }
  );
  
  // Option B: last30days (richer, but requires the skill setup)
  // Run: /last30days "${neighborhood} ${city}" 
  // Parse the output
  
  return results.data.web;
}
```

```typescript
// routes/generate.ts
export async function generateBrief(zip: string, agentName: string) {
  // Run all scrapers in parallel
  const [schools, dining, walkability, market, safety, sentiment] = 
    await Promise.all([
      getSchools(zip),
      getDining(zip),
      getWalkability(zip),
      getMarket(zip),
      getSafety(zip),
      getNeighborhoodSentiment(neighborhoodFromZip(zip), cityFromZip(zip)),
    ]);

  // Synthesize into narrative with Gemini
  const narrative = await synthesize({ schools, dining, walkability, market, safety, sentiment });

  // Generate PDF
  const pdf = await generatePDF({ zip, agentName, ...narrative });

  // Email it
  await sendEmail(agentName, pdf);
  
  return { success: true, pdfUrl: pdf.url };
}
```

## Trigger.dev (If You Add Scheduled Refreshes Later)

For V1, skip scheduling entirely. The agent clicks "Generate" and gets the brief in 30-60 seconds. No cron needed.

For V2 (if agents want weekly auto-refreshes for their active listings):

```typescript
// trigger.ts
import { schedules } from "@trigger.dev/sdk/v3";

export const weeklyBrief = schedules.task({
  id: "weekly-neighborhood-brief",
  cron: "0 8 * * 1", // Every Monday at 8am
  run: async () => {
    const activeSubscriptions = await getActiveSubscriptions();
    for (const sub of activeSubscriptions) {
      for (const zip of sub.trackedZips) {
        await generateBrief(zip, sub.agentName);
      }
    }
  },
});
```
